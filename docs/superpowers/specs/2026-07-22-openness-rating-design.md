# The Openness rating — design

**Date:** 2026-07-22
**Status:** approved, ready for planning

## The problem

The Pyatkov Index answers one direction of the question: *given a passport, how much
of the world's destination value does it open?* The inverse is unanswered. A reader who
cares about one country — where they want to move, study, or invite someone from — has no
way to ask *who can get in here, and how easily?*

This spec adds that second rating: pick a destination, see which passports give which
access to it, ordered and scored.

## The number

For destination `d`, the **openness score** is the credit-weighted share of the world's
people who can enter without a prior visa:

```
openness(d) = 100 × Σ_p credit(p → d) × pop(p) / Σ_p pop(p)
```

The sums run over all 199 passports **including `d` itself**, whose own citizens are
always admitted at credit 1.0. This mirrors the self-inclusion already used by the
passport score: numerator and denominator cover the same pool, so a destination open to
every passport scores exactly 100 and the number reads as a plain percentage.

Credits are the existing access tiers, unchanged:

| Access tier | Credit |
| --- | --- |
| Visa-free / freedom of movement | 1.0 |
| Visa on arrival / eTA | 0.8 |
| eVisa | 0.5 |
| Visa required / no admission | 0 |

Population comes from the World Bank series `SP.POP.TOTL` (latest available per country).
It is used **only** as the openness denominator. It does not enter the destination-weight
composite, which stays the four published signals (GDP, arrivals, HDI, migrant stock), so
no existing published number moves.

### The Δ column

Alongside the weighted score, each destination also gets an unweighted baseline:

```
equalScore(d) = 100 × Σ_p credit(p → d) / 199
```

Every passport counts as one. Each of the two scores is ranked independently, and
`delta = equalRank − rank` — positive means population-weighting lifts the destination.

This is the same signature move the rankings table makes with `Δ vs count`, pointed the
other way. A destination open to fifty microstates but closed to India and China falls
hard under population weighting; a destination open to China alone rises. The delta makes
the argument visible in a single number.

### Rank discipline

Ranks reuse `assignRanks` from `pipeline/scores.ts`:

- **Dense ranking** (1-2-2-3), matching the passport table.
- The weighted rank keys on the score **as displayed** (one decimal), so destinations the
  reader sees as equally open share a rank.
- The equal-weight rank keys on exact integer tenths of credit, never the float score, so
  mathematically tied destinations are never split by 1-ulp drift (the B1 discipline).

## Pipeline changes

| File | Change |
| --- | --- |
| `pipeline/fetch.ts` | Save `SP.POP.TOTL` to `data/raw/population.json` via the existing `WB_LATEST` helper. Export the indicator id for regression tests. |
| `pipeline/signals.ts` | `Override` gains an optional `population`. New `loadPopulations(iso3List, source, overrides)` returns `Map<iso3, number>` and **throws** listing every country with no population — the denominator must cover all 199 or the score is a lie. |
| `pipeline/openness.ts` (new) | `computeOpenness(matrix, populations, names)` → `DestinationOpenness[]`, sorted by rank. |
| `pipeline/types.ts` | New `DestinationOpenness` interface. |
| `pipeline/build.ts` | Read `population.json`, compute openness, write `site/src/data/openness.json`. Add a `population` vintage entry. Extend `sanityGuards`. |
| `pipeline/data-drift.ts` | `openness.json` joins `DATA_FILES`. |

`TWN` and `VAT` are absent from the World Bank population series and get researched
`population` values in `data/raw/manual-overrides.json`, alongside their existing
entries, each with a cited `source`.

### New sanity guards

The build fails rather than publishing corrupted numbers:

1. Every openness score is finite and within `[0, 100]`.
2. Face validity: the ten most open destinations must include known broadly-visa-free
   destinations, not noise.
3. Face validity, other end: the least open destinations must include known
   near-universally-visa-requiring destinations.

### Output shape

`site/src/data/openness.json`:

```json
{
  "destinations": [
    {
      "iso3": "...", "iso2": "...", "name": "...",
      "score": 0, "rank": 1,
      "equalScore": 0, "equalRank": 1, "delta": 0,
      "counts": { "visa-free": 0, "visa-on-arrival": 0, "e-visa": 0, "visa-required": 0 }
    }
  ]
}
```

`counts` covers all 199 passports, with the destination's own passport counted visa-free.

## Site changes

### `/destination/[iso3]/` — 199 static pages

The answer to "choose a country, see which passports give which access to it". One page
per destination, generated by `getStaticPaths` over `openness.json`, matching the existing
`/passport/[iso3]` pattern: linkable, indexable, and working without JavaScript.

- **Hero**: flag + destination name.
- **Stats**: Openness score, Openness rank, Count rank, Δ, and the destination's own
  weight in the index with its weight rank.
- **Where the openness comes from**: a tier × passports × points table, with the points
  reconciled by the existing `reconcilePoints` so the rows sum exactly to the displayed
  score — the same treatment the passport page gives its tier table.
- **The main table**: all 199 passports with their access tier and their own index score
  and rank. Default order is the visa-free block first (descending credit), and within
  each tier by the passport's Pyatkov score. Filterable by country name/code and sortable
  by column, reusing the rankings table's script pattern and `searchIndex`.

### `/openness/` — the rating

All 199 destinations ranked by openness score, in the shape of the rankings table:
`#`, destination, score, Δ vs count, count rank. Each row links to its destination page.
Filterable and sortable. Added to the site nav.

### Existing pages

- `Base.astro`: new nav entry between Rankings and Destinations.
- `destinations.astro`: each destination name links through to its new page.
- `methodology.astro`: a section deriving the openness score, its self-inclusion, its
  population denominator, and the Δ column.
- `README.md`: the same, in prose, with the concrete numbers the shipped data produces.

## Testing

Test-first throughout.

**`pipeline/__tests__/openness.test.ts`** (new)

- Credit weighting: a destination visa-free to a large-population passport scores above
  one visa-free only to a small-population passport.
- Self-inclusion: the destination's own population is counted at credit 1.0.
- A destination visa-free to every passport scores exactly 100.
- A destination visa-required to every foreign passport scores exactly its own population
  share.
- Graded credit: visa-on-arrival earns 0.8, eVisa 0.5, visa-required 0.
- Dense, tie-safe ranks on both the weighted and equal-weight scores.
- `delta = equalRank − rank`, with a case where population weighting demonstrably moves a
  destination.
- `counts` sums to the passport count, with self counted visa-free.
- Throws when a passport in the matrix has no population.

**`pipeline/__tests__/signals.test.ts`** (extended)

- `loadPopulations` prefers an override over the World Bank value.
- `loadPopulations` throws naming every country with no population.

**`pipeline/__tests__/build.test.ts`** (extended)

- The new sanity guards throw on out-of-range and face-invalid openness data.

**`pipeline/__tests__/fetch.test.ts`** (extended)

- The population URL is the expected World Bank endpoint and series.

**`pipeline/__tests__/readme-claims.test.ts`** (extended)

- Any concrete openness number the README cites is asserted against the shipped data.

## Out of scope

- Changing the destination weights or any published passport number.
- Historical or time-series openness.
- Reciprocity analysis (who lets in whom, mutually).
